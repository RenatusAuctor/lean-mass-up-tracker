# Agent Rules for Lean Mass Up Tracker

## Deployment Automation
- **Auto-Commit & Push**: Whenever you modify code or files in this workspace (HTML, CSS, JS, etc.), you MUST automatically stage the changes (`git add .`), commit them with a descriptive message (`git commit -m "..."`), and push them to the remote repository (`git push`) to ensure GitHub Pages is always up to date. You do not need to ask the user for permission to push if you just successfully completed a feature they requested.
